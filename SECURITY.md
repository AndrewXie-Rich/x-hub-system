# Security Policy

## Supported Versions

Security fixes are provided for:
- the default branch (development)
- the most recent tagged release (when releases exist)

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security reports.

Use this repository's **Security Advisories** flow ("Report a vulnerability"). If that option is not available, open an issue with *minimal details* and request a private channel.

## What to Include

- Affected component(s) (X-Hub app / gRPC server / runtime / connector / skill runner)
- Version / commit hash
- Reproduction steps (ideally a minimal PoC)
- Impact (what an attacker can do)
- Any proposed mitigation or patch

## Response Targets

- Acknowledge receipt: within 7 days
- Provide an initial assessment / next steps: within 14 days
