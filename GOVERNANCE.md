# Governance

This document explains how repository stewardship and contributions are handled
for X-Hub System.

It is a project-governance document. It is not a separate software license and
does not override `LICENSE`, `LICENSE_POLICY.md`, or `TRADEMARKS.md`.

## Current Model

- The repository is currently open source under the MIT License.
- The project owner retains control over release scope, roadmap, branding, and
  trademark use.

## Contributions

External contributions are welcome.

Contributions are reviewed based on:

- technical quality
- security and trust-boundary impact
- scope alignment with the validated public mainline
- maintenance cost
- documentation and test coverage when behavior changes

Acceptance of a contribution does not create maintainer, ownership, or
governance rights for the contributor beyond the rights granted by the
repository license and applicable law.

## Stewardship

The project owner may:

- accept or reject contributions
- define release boundaries
- revise governance processes

Those decisions should be communicated clearly in top-level repository
documentation when they materially affect contributors or adopters.
